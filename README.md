# GOYO - 감정 운영 인텔리전스

케어 환경의 감정 운영 인텔리전스 데스크톱 애플리케이션

## 프로젝트 개요

GOYO는 심리상담 원리를 돌봄 환경 운영 의사결정에 적용하여 케어 인력 번아웃을 예방하는 감정 운영 인텔리전스 레이어입니다.

### 핵심 가치

> "감정을 측정할 수 있다면, 관리할 수 있습니다. 관리할 수 있다면, 개선할 수 있습니다."

GOYO는 위기가 되기 전에 감지하고, 데이터로 증명하며, 행동을 촉구합니다.

## 주요 기능

### 📊 감정 모니터링 대시보드
- 실시간 케어 인력 감정 상태 추적
- 팀별 감정 분포 시각화
- 최근 7일 감정 트렌드 분석

### 🔔 번아웃 리스크 알림
- 실시간 리스크 감지 시스템
- 고위험군 조기 경보
- 부정적 감정 증가율 추적 (85% 증가 감지 시 알림)
- 번아웃 리스크 점수 계산 (72% 이상 고위험)

### 💡 개입 방안 제안
GOYO의 핵심 기능으로, 4단계 구조화된 개입 방안을 제공:

1. **즉시 실행** (24시간 이내)
   - 관리자와 1:1 면담 일정
   - 비공식적 대화로 감정 상태 파악

2. **단기 조치** (1주일 이내)
   - 업무량 재조정
   - 감정적 부담 케이스 분산

3. **중기 지원** (2-4주)
   - 전문 상담 서비스 연결
   - EAP 프로그램 안내

4. **지속 모니터링**
   - 2주간 감정 패턴 추적
   - 데이터 기반 개선 여부 확인

### 👥 케어 인력 관리
- 개인별 감정 이력 추적
- 30일 감정 데이터 분석
- 상세 감정 로그 조회

### 📈 운영 인사이트 리포트
- Emotional Operations Framework 소개
- 데이터 기반 의사결정 지원
- 핵심 인사이트 제공

## 기술 스택

- **Electron** - 크로스 플랫폼 데스크톱 애플리케이션
- **Chart.js** - 감정 데이터 시각화
- **SQLite (better-sqlite3)** - 로컬 데이터베이스
- **Vanilla JavaScript** - 순수 JavaScript로 구현

## 설치 및 실행

### 요구사항
- Node.js 16 이상
- npm 또는 yarn

### 설치
```bash
npm install
```

### 실행
```bash
# 프로덕션 모드
npm start

# 개발 모드 (개발자 도구 포함)
npm run dev
```

## 데이터베이스 스키마

### care_workers (케어 인력)
- id, name, role, team, hire_date, status

### emotion_logs (감정 로그)
- id, worker_id, timestamp, emotion_type, intensity, context

### risk_alerts (리스크 알림)
- id, worker_id, alert_date, risk_score, risk_level, status

### interventions (개입 조치)
- id, alert_id, intervention_type, description, deadline, status, completed_date

## 샘플 데이터

애플리케이션은 시연을 위한 샘플 데이터를 자동으로 생성합니다:
- 5명의 케어 인력 (김미영, 이정수, 박서연, 최민준, 정수진)
- 각 인력당 30일간의 감정 로그
- 김미영(워커 1번)은 고위험군으로 설정되어 번아웃 리스크 알림 발생

## 시연회 준비사항

### 핵심 시연 시나리오

1. **대시보드 개요** (30초)
   - 전체 케어 인력 현황
   - 고위험 알림 1건 표시
   - 감정 분포 및 트렌드 차트

2. **리스크 알림** (1분)
   - 김미영님 고위험 알림 클릭
   - 85% 부정적 감정 증가율 강조
   - 72% 번아웃 리스크 점수 설명
   - 3일 내 임계점 도달 경고

3. **개입 방안** (2분) - **클라이맥스**
   - "개입 방안 보기" 버튼 클릭
   - 4단계 구조화된 액션 플랜 제시
   - 즉시 실행 → 단기 → 중기 → 모니터링 흐름 설명
   - 핵심 메시지: "감정 리스크를 줄이는 것은 곧 운영 비용을 줄이는 것입니다"

4. **케어 인력 상세** (1분)
   - 김미영님 프로필 클릭
   - 30일 감정 이력 차트 분석
   - 감정 로그 세부 내역

5. **운영 인사이트** (1분)
   - Emotional Operations Framework 소개
   - GOYO의 차별화 포인트 강조

## 핵심 포지셔닝

GOYO는 독립형 서비스가 아니라 **기존 케어 시스템에 통합되는 인텔리전스 레이어**입니다.

- ❌ 상담 서비스를 제공하는 것이 아님
- ✅ 상담 원리 기반의 운영 인사이트를 제공

## AI 아키텍처

현재 Decision Intelligence(지원형 AI)로 작동하지만, Agentic AI로 확장 가능한 구조:
- 목표 지향적: 번아웃 예방
- 자율 감지: 실시간 감정 신호 모니터링
- 맥락 해석: 상황 기반 패턴 분석
- 행동 제안: 구체적인 개입 방안
- 학습 루프: 피드백 기반 개선
